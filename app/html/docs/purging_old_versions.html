Purging Old Versions

During the alpha phase of Package Control 2.0, quite a number of users installed Package Control via git. However, with version 2.0.0-alpha7 there was a bug that prevented automatically upgrading Package Control itself. Because of this, there are quite a number of users still running old versions of Package Control. These instructions will step through purging the old version of Package Control and upgrading to a stable release.

  2. Delete the folder named Package Control
  3. Browse up a folder and then into Installed Packages
  4. Delete Package Control.sublime-package if it exists
  5. Reinstall Package Control using the installation instructions
